# 历史归档区说明

`历史归档区/` 只用于保存**已经退出当前主链**、但仍有追溯价值的历史材料。

## 这里放什么

- 历史规划文件、阶段性分析稿、旧版审计稿
- 已废弃源码备份
- 外部参考资料快照
- 历史打包归档与本地工具痕迹

## 这里不放什么

- 当前系统运行依赖的代码
- 当前 Web / CLI / 训练主链所需配置
- 当前 runtime 运行现场
- 仍会被主流程 import 的模块

## 使用原则

- 归档内容默认**不参与当前功能实现**。
- 若某个历史文件仍值得参考，应该在 `docs/` 中说明“参考价值”和“与当前实现的差异”，而不是把它重新当成主链文档。
- 如果未来再次做大规模重构，建议继续把被替换掉的方案、计划稿和临时分析稿收口到这里，而不是散落回仓库根目录。
